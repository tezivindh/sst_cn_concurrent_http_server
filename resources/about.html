Project Overview

This multi-threaded HTTP server is a comprehensive implementation of a web server built from scratch using low-level socket programming. It demonstrates deep understanding of network protocols, concurrent programming, and security best practices.

Key Features

  • Thread Pool Architecture: Configurable thread pool with up to 10 concurrent workers
  • HTTP/1.1 Support: Full implementation of HTTP protocol with persistent connections
  • Binary File Transfer: Efficient handling of images, documents, and other binary data
  • JSON API: POST endpoint for receiving and storing JSON data
  • Security Hardening: Path traversal protection and host header validation
  • Connection Management: Keep-alive support with configurable timeouts
  • Comprehensive Logging: Detailed request/response tracking with timestamps

Technical Implementation

The server is built using Python 3 with the standard library's socket module for low-level network communication. Threading is managed through a custom thread pool implementation that efficiently distributes incoming connections across worker threads.

Architecture Highlights

Socket Layer: TCP sockets with proper lifecycle management
Thread Management: Fixed-size thread pool with connection queue
Request Parsing: Complete HTTP request parser with header extraction
Response Building: RFC-compliant HTTP responses with proper headers
Security: Input validation and path canonicalization

Supported Operations

GET Requests: Serve HTML files for browser rendering, and binary files (images, text files) with Content-Disposition headers for download functionality.

POST Requests: Accept JSON payloads, validate the data, and store them in the uploads directory with timestamped filenames.

Performance Characteristics

The server can handle up to 10 concurrent connections simultaneously, with additional connections queued (up to 50 in the listen queue). Each persistent connection supports up to 100 requests with a 30-second timeout between requests.
